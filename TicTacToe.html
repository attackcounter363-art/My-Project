Tic Tac Toe: From Rules to Optimal Play

This page explains different implimentations of Tic Tac Toe, from human-vs-human to AI using minimax

Overview

Tic Tac Toe is a simple turn-based game that provides a greate environment to explore different decision-making stratagies.

Game Rules

  • Two player alternate turns
  • Players place X or O on a 3x3 grid
  • First to align symbols wins

2 Player Mode

In this mode, two humans play against each other with no AI involvement.

Random AI

The first attempt at automation was deliberately naïve.

Instead of asking the computer to “play well”, I asked it to simply play legally. At each turn, I selected one of the available empty cells at random.

This immediately solved one problem: the game could now be played by a single human against the computer.

However, it exposed another.

The computer has no understanding of the board. It missed obvious wins, failed to block threats, and often made moves that were clearly harmful.

This made one thing clear: automation is not the same as strategy.

Choosing a move is easy. Choosing a good move is not.

This idea can be seen directly in the move-selection function below:-


import random

def get_random_move(board):
    """
    Selects a move uniformly at a random from all currently available positions.

    This function demonstrates the simplest form of automated play.
    It guarentees legal moves, but does not evaluate the board state or anticipate future outcomes.
    """
    empty_cells = [
        (r,c)
        for r in range(3):
        for c in range(3):
        if board[r][c] == ""
    ]

    return random.choice(empty_cells)

Rule-Based AI

address the randomness, I introduced simple rules.

  1. If a winning move exists, the computer would take it.
  2. If the opponent was about to win, the computer would block.
  3. If neither applied, it would fall back to positional preferences.

This approach worked surprising well. The computer now appeared intentional and reactive. Against casual play, it perform convincingly. But after extended testing, a deeper flaw emerged. The rules only considered the current board. They could not anticipate situations where every immediate move looked acceptable, but all of them led to a loss a few turns later. At this point, the limitation was no longer about what the computer could see - it was about how far it could see.

This idea can be seen directly in the move-selection function below:-

Minimax

By the time the rule-based strategy was in place, the computer appeared competent. It could win simple games, block obvious threats, and generally behave in a way that felt intentional. However, longer games revealed a persistent issue.

There were positions where none of the immediate moves looked dangerous, yet the game was already lost.

The rule-based player could not recognize these situations because every rule it followed examined only the current board. Improving individual rules helped omly marginally; the problem was more fundamental.

At this point, it became clear that choosing a move required more than reacting to the present.

What was needed was a way to reason about how the game would unfold after a move was made.

This led to the use of minimax strategy.

Instead of asking “Which move looks good right now?”, the computer now asks a different question:

“is I makethis move, and my opponent explores the game forward in time.”

To answer this, the program explores the game forward in time.

From the current board, each legal move produces a new board state. From those states, further moves are possible, and the process continues until the game must end - either in a win, a loss, or a draw.

These final outcomes are treated as evaluations:

  1. A win us desirable,
  2. A loss is undesirable,
  3. And a draw lies in between.

The result is no longer a logically reasonable move, but a globally optimal one.

The result is no longer a logically reasonable move, but a globally optimal one. Wit minimax, the computer no longer relies on heuristics or chance.

If a winning path exists, it will find it. If no such path exists, it will avoid losing altogether.

This strategy reaches the theoretical limit of Tic-Toc-Toe.Any further improvement would not come from better decision making, but from changing the game itself.

This idea can be seen directly in the move-selection function below:-

    Recursively evalutaes the game tree strategy explores future board states.

    This allows the computer to avoid short-sighted moves that appear safe now but lead to inevitable loss later.
    """
